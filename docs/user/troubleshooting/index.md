# 故障排查

FenixAgent 使用中遇到问题？这里有一些解决方案。


## 快速诊断

遇到问题时，按以下步骤排查：

1. 刷新页面，看问题是否消失
2. 记录完整的错误提示（截图更好）
3. 根据问题类型跳转到下方对应章节
4. 如果还是解决不了，联系管理员


## 大模型配置问题

### 测试连接失败

点击「测试」按钮后显示错误。

**可能原因**：
- API Key 不正确或已过期
- Base URL 填写错误
- AI 服务商暂时不可用

**解决方法**：
1. 重新检查 API Key 是否正确（注意前后空格）
2. 确认 Base URL 格式（一般以 `/v1` 结尾）
3. 稍后再试，可能是服务商临时故障

### API Key 获取地址

| 服务商 | 地址 |
|--------|------|
| OpenAI | https://platform.openai.com/api-keys |
| Anthropic | https://console.anthropic.com/settings/keys |
| 阿里云百炼 | https://bailian.console.aliyun.com/ |

### 保存后 API Key 变成 `{env:xxx}` 格式

这是正常的安全机制，实际密钥已安全存储，不影响使用。编辑时 API Key 留空表示不修改。

### 模型列表为空

点击「测试」后没有显示模型。

**可能原因**：
- API Key 权限不足（如免费账号可能只支持部分模型）
- 服务商 API 地址不正确

**解决方法**：确认 API Key 对应的账号有权限访问目标模型，也可以手动点击「+ 添加模型」逐个添加。


## Agent 问题

### Agent 显示离线

Agent 列表中的 Agent 状态显示为「离线」。

**可能原因**：
- 对应的运行环境未启动
- 网络连接中断

**解决方法**：
1. 确认运行环境是否已启动（联系管理员确认）
2. 刷新页面重新加载状态
3. 如果持续离线，联系管理员检查后台服务

### Agent 权限不生效

Agent 拒绝执行你允许的操作。

**解决方法**：
1. 进入 Agent 编辑页面，检查「权限配置」标签页
2. 确认对应工具的权限设为「允许」而不是「询问」或「拒绝」
3. 检查是否有通配符规则覆盖了你的设置（如 `*.env: deny` 会拒绝所有 .env 文件的操作）

### 内置 Agent 找不到了

`build`、`plan`、`general` 等系统 Agent 不在列表中。

内置 Agent 不可删除。如果突然消失，联系管理员检查配置文件是否损坏。


## 会话问题

### 会话列表为空

Sessions 页面没有显示任何会话。

**解决方法**：
1. 确认至少有一个 Agent 处于在线状态
2. 检查页面顶部的筛选条件是否过滤掉了所有会话
3. 尝试新建一个会话

### 文件上传失败

**解决方法**：
1. 确认文件格式受支持（PDF、Markdown、TXT、源代码文件等）
2. 文件大小建议不超过 10MB
3. 刷新页面后重新上传

### 会话刷新后丢失

**解决方法**：
1. 检查对应的 Agent 是否在线
2. 清除浏览器缓存后重新登录
3. 如果会话确实丢失，新建会话即可


## 定时任务问题

### Cron 表达式不生效

任务创建成功但从未执行。

**解决方法**：
1. 确认表达式格式为 5 个字段：`分 时 日 月 周`
2. 使用页面上的快捷选择验证（如「每小时」= `0 * * * *`）
3. 可以在 [crontab.guru](https://crontab.guru/) 验证表达式含义

### 任务显示「已跳过」

**解决方法**：对应的运行环境未就绪，确认环境在线后等待下一次调度。

### 立即执行无响应

**解决方法**：
1. 确认运行环境在线
2. 检查任务内容描述是否清晰
3. 如果任务超时，编辑任务增加超时时间（最大 180 分钟）


## 知识库问题

### 文档上传失败

**解决方法**：
1. 确认文件格式受支持（PDF、TXT、MD、DOCX 等）
2. 文件大小建议不超过 50MB
3. 上传后显示「失败」状态，查看资源卡片中的错误信息

### Agent 检索不到知识库内容

**解决方法**：
1. 确认知识库状态为「成功」，不是「处理中」或「失败」
2. 确认 Agent 已绑定该知识库（Agent 编辑 → 知识库标签页）
3. 处理中的资源需要等待后台完成索引


## Skills 问题

### 技能创建后 Agent 无法使用

**解决方法**：检查全局技能是否被禁用（工作区技能始终可用，不需要启用）。

### 上传提示「缺少 SKILL.md」

**解决方法**：确保每个子目录下都有 `SKILL.md` 文件（大写）。

### 工作区技能显示「扫描超时」

**解决方法**：联系管理员检查项目路径是否可访问。


## MCP 问题

### MCP 服务器「连接失败」

**解决方法**：
1. Remote 类型：点击「测试」按钮检查 URL 是否可达
2. Local 类型：联系管理员确认命令行工具已正确安装
3. 检查环境变量是否正确填写

### 检测后工具数量为 0

**解决方法**：
1. 尝试重新检测
2. 增加「超时时间」设置
3. 确认 MCP 服务器版本是否兼容


## 编排问题

### 页面显示「连接失败」

**解决方法**：
1. 点击页面上的「重试」按钮
2. 如果持续失败，联系管理员检查编排服务是否正常运行


## 登录问题

### 无法登录控制面板

**解决方法**：
1. 确认用户名和密码正确
2. 清除浏览器 Cookie 后重试
3. 联系管理员重置密码

### API Key 无效

**解决方法**：
1. 前往「API Key」管理页面确认密钥状态
2. 如果密钥已过期或失效，重新生成
3. 确认使用格式：`Bearer rcs_xxx`


## 联系管理员

以上方法都无法解决时，请联系管理员，提供以下信息：

1. 遇到问题时的页面截图
2. 完整的错误提示内容
3. 问题发生的时间和操作步骤

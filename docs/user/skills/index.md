# Skills 管理

Skills 是自定义技能模块，用 Markdown 格式编写，让 Agent 学会特定任务。点击左侧导航栏的 **Skills** 进入管理页面。


## 技能列表

页面按来源分组显示技能：

### 来源类型

| 来源 | 说明 | 存储位置 |
|------|------|----------|
| 全局 | 所有 Agent 共享 | `~/.agents/skills/` |
| 工作区 | 当前项目专属 | `<项目>/.agents/skills/` |

### 技能信息

展开来源行后可以看到该来源下的所有技能。工作区技能会显示项目名称标识。


## 创建技能

点击 **+ 新建技能** 或 **上传技能** 按钮，对话框有两个标签页：

### 创建技能

填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 技能名称 | 唯一标识符，建议 kebab-case | `code-review` |
| 描述 | 简要说明用途 | `对代码进行系统性审查` |
| 内容 | Markdown 格式的指令 | 见下方示例 |

### 上传技能

1. 选择包含技能的文件夹
2. 系统自动识别每个子目录为一个技能
3. 显示预览列表，标注可导入的技能和缺少 SKILL.md 的目录
4. 支持冲突处理：跳过或覆盖已有技能


### 技能内容示例

```markdown
你是一位代码审查专家。

## 审查要点

1. **代码风格**：检查是否符合项目规范
2. **潜在 bug**：查找可能导致错误的代码
3. **性能问题**：识别性能瓶颈
4. **安全性**：检查是否存在安全漏洞

## 输出格式

## 发现的问题

### [严重级别] 问题标题
- **位置**：文件路径:行号
- **说明**：问题描述
- **建议**：修复建议

## 总结
- 总问题数：X
- 严重：Y
- 一般：Z
```


### 技能包结构

```
skills-package/
├── skill-a/
│   └── SKILL.md      # 必需
├── skill-b/
│   ├── SKILL.md
│   └── assets/       # 可选
└── skill-c/
    └── SKILL.md
```

每个技能目录必须包含 `SKILL.md` 文件。

## 启用/禁用技能

只有**全局技能**支持启用/禁用。工作区技能始终可用。

点击全局技能的 **启用** 或 **禁用** 按钮即可切换状态。

## 编辑技能

点击 **编辑** 按钮，修改后保存。

编辑工作区技能时会弹出确认提示，因为修改直接影响项目文件。

## 删除技能

点击 **删除** 按钮，确认后即可删除。工作区技能会删除项目文件，无法恢复。

## 批量操作

勾选多个技能后，批量操作栏支持**批量删除**。

## 搜索

搜索框支持：
- 按技能名称搜索
- 按描述内容搜索
- 按来源名称过滤


## 常见问题

### 技能创建后 Agent 无法使用？

检查全局技能是否被禁用。工作区技能无需启用。

### 工作区技能显示"扫描超时"？

检查项目路径是否存在，FenixAgent 是否有读取权限。

### 上传时提示"缺少 SKILL.md"？

确保每个子目录都有 `SKILL.md` 文件（大写）。

### 技能内容支持什么格式？

标准 Markdown 语法：标题、列表、代码块、粗体、斜体、链接等。

### 如何从其他项目导入技能？

复制 `.agents/skills/` 目录到新项目，或使用上传功能。

### 全局技能 vs 工作区技能

| 对比项 | 全局技能 | 工作区技能 |
|--------|----------|------------|
| 可用范围 | 所有 Agent | 仅当前项目 |
| 存储位置 | `~/.agents/skills/` | `<项目>/.agents/skills/` |
| 版本控制 | 不推荐 | 推荐 |

### 技能命名规范

建议使用 kebab-case（如 `code-review`），避免中文或特殊字符。

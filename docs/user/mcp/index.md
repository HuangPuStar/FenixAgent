# MCP 服务器

MCP (Model Context Protocol) 让 AI Agent 连接外部工具和数据源，比如数据库查询、API 调用、文件操作等。

## 访问 MCP 管理

1. 登录 FenixAgent 控制面板
2. 点击左侧导航栏的 **MCP**

列表显示已配置的 MCP 服务器：

| 列 | 说明 |
|----|------|
| 名称 | MCP 服务器的唯一标识符 |
| 类型 | Local（本地启动）或 Remote（URL 连接） |
| 状态 | 已启用 / 已禁用 |
| 命令/URL | 启动命令或连接地址 |
| Tools | 已发现的工具数量 |

## Local 类型

Local 类型通过命令行在本地启动 MCP 服务器进程，适合使用 npm 包或本地可执行文件。

### 新建 Local 服务器

1. 点击 **新建服务器**
2. 类型选择 **Local（命令行启动）**
3. 填写表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 唯一标识符，创建后不可修改 | `github-mcp` |
| 命令 | 启动命令，空格分隔 | `npx -y @modelcontextprotocol/server-github` |
| 环境变量 | 命令所需的环境变量 | `GITHUB_TOKEN` 等 |
| 超时时间 | 毫秒，留空使用默认值 | `5000` |

4. 点击 **保存**

### GitHub MCP Server 示例

| 字段 | 填写内容 |
|------|----------|
| 名称 | `github` |
| 类型 | Local |
| 命令 | `npx -y @modelcontextprotocol/server-github` |
| 环境变量 | `GITHUB_TOKEN` = `ghp_xxxxx...` |

需要先在 GitHub 申请 Personal Access Token。

### 命令格式

命令字符串按空格拆分为数组。参数包含空格时用双引号包裹：

```bash
# 正确
npx -y @modelcontextprotocol/server-github --arg1 "value with spaces"

# 错误（缺少引号）
npx -y @modelcontextprotocol/server-github --arg1 value with spaces
```

## Remote 类型

Remote 类型通过网络 URL 连接到已运行的 MCP 服务器，适合连接云端部署或第三方提供的 MCP 服务。

### 新建 Remote 服务器

1. 点击 **新建服务器**
2. 类型选择 **Remote（URL 连接）**
3. 填写表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 唯一标识符 | `my-remote-mcp` |
| URL | MCP 服务器地址 | `https://api.example.com/mcp` |
| 请求头 | HTTP 请求头 | `Authorization: Bearer xxx` |
| OAuth 配置 | OAuth 认证配置（可折叠面板） | Client ID、Secret 等 |
| 超时时间 | 毫秒，留空使用默认值 | `5000` |

4. 点击 **测试** 验证连接（可选）
5. 点击 **保存**

### Remote 配置示例

| 字段 | 填写内容 |
|------|----------|
| 名称 | `company-mcp` |
| 类型 | Remote |
| URL | `https://mcp.example.com/sse` |
| 请求头 | `Authorization` = `Bearer rc_xxx...` |

### 测试连接

填写 URL 后点击 **测试**：

- **连接成功**：显示服务器名称、版本和工具数量
- **可达但非 MCP**：服务器可访问但未返回 MCP 协议响应
- **连接失败**：显示错误信息

### 环境变量和请求头

- 点击 **+ 添加** 增加新行
- 点击 **删除** 移除某一行

## 启用/禁用

### 单个操作

在服务器列表的操作列：

- 点击 **启用** 激活服务器
- 点击 **禁用** 停用服务器

### 批量操作

1. 勾选多个服务器
2. 在批量操作栏中选择：
   - **批量启用**
   - **批量禁用**
   - **批量删除**

## 发现工具

配置完成后需要检测来发现可用工具。

### 检测

1. 在列表中找到目标服务器
2. 点击 **检测** 按钮
3. 完成后显示发现的工具数量

### 查看工具详情

展开服务器行查看已发现的工具：

- **工具名称** — MCP 工具的标识符
- **描述** — 工具功能说明
- **参数** — 点击「参数」查看输入 JSON Schema（使用 HTML `<details>` 元素可折叠）

## 编辑

1. 点击 **编辑** 按钮
2. 修改配置（类型和名称不可修改）
3. 点击 **保存**

修改类型需要删除后重新创建。

## 删除

### 单个删除

1. 点击 **删除** 按钮
2. 确认操作

### 批量删除

1. 勾选多个服务器
2. 点击 **批量删除**
3. 确认操作

## 常见问题

### MCP 服务器显示「连接失败」

**可能原因**：
- Remote URL 地址错误或服务不可用
- 命令行启动的 MCP 服务器未正确安装
- 环境变量配置错误（如 API Key 无效）

**解决方法**：
1. Remote 类型：用「测试」按钮验证 URL 可达性
2. Local 类型：在终端手动运行命令，检查是否正确启动
3. 检查环境变量是否正确

### 检测后工具数量为 0

**可能原因**：
- MCP 服务器未正确实现协议
- 服务器启动但未暴露工具
- 网络连接超时

**解决方法**：
1. 检查 MCP 服务器日志
2. 增加超时时间配置
3. 确认 MCP 服务器版本是否兼容

### `npx -y` 是什么

`npx -y` 是 npm 包执行器的参数：
- `npx` — 执行 npm 包而不需要全局安装
- `-y` — 跳过确认提示，自动同意安装

确保命令在自动化环境中正常运行。

### Agent 可以直接使用 MCP 工具吗

可以。配置并启用 MCP 服务器后，Agent 会自动发现并调用这些工具，在会话中直接使用 MCP 提供的能力，无需额外配置。

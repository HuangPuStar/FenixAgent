# 定时任务

定时任务按 Cron 表达式自动执行 Agent 任务。

打开方式：导航栏点击「任务」，或访问 `/ctrl/tasks`。

## 新建任务

点击右上角「新建任务」。

**必填**：

- **名称**：任务唯一标识
- **Cron 表达式**：执行时间配置
- **Environment**：运行环境
- **任务内容**：自然语言描述要做什么

**选填**：

- **描述**：说明用途
- **超时时间**：1-180 分钟，默认 30 分钟
- **启用状态**：默认启用

### Cron 表达式

格式：`分 时 日 月 周`

```
* * * * *
│ │ │ │ │
│ │ │ │ └─ 星期 (0-6, 0=周日)
│ │ │ └─── 月份 (1-12)
│ │ └───── 日期 (1-31)
│ └─────── 小时 (0-23)
└───────── 分钟 (0-59)
```

快捷选择下拉框提供 5 个预设：

| 预设 | 表达式 |
|------|--------|
| 每 5 分钟 | `*/5 * * * *` |
| 每小时 | `0 * * * *` |
| 每天早 9 点 | `0 9 * * *` |
| 工作日早 9 点 | `0 9 * * 1-5` |
| 每月 1 号 | `0 0 1 * *` |

特殊字符：

- `*`：任意值
- `,`：枚举（`1,3,5`）
- `-`：范围（`1-5`）
- `/`：间隔（`*/5` 每 5 个单位）

在 [crontab.guru](https://crontab.guru/) 验证表达式。

### 任务内容示例

```
检查 /workspace/src 目录下的所有 .ts 文件，找出未使用的 imports 并输出到 report.md
从 API 获取最新数据，更新 data.json 文件，并发送通知
运行测试套件，将结果写入 test-results/ 目录
```

## 任务列表

列表支持搜索、排序和筛选。

| 列 | 说明 |
|----|------|
| 名称 | 任务名称 |
| Cron 表达式 | 调度时间配置 |
| Environment | 关联的运行环境 |
| 状态 | 已启用 / 已禁用 |
| 上次执行 | 最近一次执行时间 |
| 下次执行 | 下一次计划执行时间 |
| 最近结果 | 最近一次执行的结果状态 |

### 操作按钮

每行右侧有编辑、日志、立即执行、启用/禁用、删除按钮。

## 执行历史

点击「日志」按钮查看。

执行记录显示格式：`时间 · 触发者 · 耗时`

触发者：scheduler（定时）或 manual（手动）。

状态：成功 / 失败 / 超时 / 已跳过。

### 查看目录

点击记录中的「查看目录」，浏览该次执行产生的文件。

右侧面板显示文件列表，包括文件名、路径、类型和大小。

### 分页

使用「上一页」和「下一页」切换，每页 20 条。

### 清空日志

点击「清空日志」删除该任务的所有历史记录（需确认）。

## 失败处理

任务失败不会自动重试，需要手动触发。

常见原因：

- **Environment 离线**：先启动 Agent
- **超时**：增加超时时间或优化任务
- **权限不足**：检查 Agent 权限配置
- **描述不清**：改写任务描述

## 常见问题

### Cron 不生效

检查表达式格式（必须 5 个字段），用快捷选择验证。

### 显示"已跳过"

Environment 未就绪，确保在线。

### 立即执行无响应

检查 Environment 是否在线，查看浏览器控制台。

### 任务超时

编辑任务，增加超时时间（最大 180 分钟）。

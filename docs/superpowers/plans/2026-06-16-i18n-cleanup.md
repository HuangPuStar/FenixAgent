# i18n 规范化 & JSON 膨胀治理 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 消除前端 51 个文件中的硬编码中文字符串，并精简 i18n JSON 中的重复 key。

**Architecture:** 两层修复——首先扩充 `common.json` 和 `components.json` 的 key 池，然后逐文件替换硬编码字符串为 `t()` 调用。JSON 膨胀治理通过合并命名空间内重复 key 和提取跨命名空间公共词汇完成。

**Tech Stack:** react-i18next, TypeScript, React

---

## 背景综述

审查发现 264 个前端源文件中：
- 140 个已使用 i18n，其中 **51 个同时存在硬编码中文**
- 124 个完全未引入 i18n（大部分是纯逻辑/utils/UI 基础组件，无需 i18n）
- 硬编码中文主要集中在：config 通用组件、chat 组件、agent-panel 页面、workflow 组件

i18n JSON 总计 **2007 个 key**（116KB），`components.json` 和 `workflows.json` 存在大量重复 key。

```typescript
export function getBadgeVariant(status: string): string {
  const map: Record<string, string> = {
    configured: "green",
    enabled: "green",
    unconfigured: "secondary",
    disabled: "secondary",
    builtIn: "blue",
    custom: "outline",
  };

}
```

将 `StatusBadge` 组件改为：
```typescript
export function StatusBadge({ status }: StatusBadgeProps) {
  const { t } = useTranslation("components");
  const variant = getBadgeVariant(status);
  const badgeVariant = variantMap[variant] || "outline";
  const isActive = status === "enabled" || status === "configured";

  const label = t(`statusBadge.${status}`, status);

  return (
    <Badge
      variant={badgeVariant}
      className={cn(
        variant === "green" && "bg-green-100 text-green-700 dark:bg-green-900 dark:text-green-300",
        variant === "blue" && "bg-blue-100 text-blue-700 dark:bg-blue-900 dark:text-blue-300",
        isActive && "status-badge-active",
      )}
    >
      {label}
    </Badge>
  );
}
```

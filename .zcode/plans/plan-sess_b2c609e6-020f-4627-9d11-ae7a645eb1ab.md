## 优化模型测试通知交互体验

### 问题分析
当前在模型卡片（左侧）点击"测试"按钮后，结果通过右上角 sonner toast 弹出，用户视线距离远，极易忽略。

### 解决方案：卡片内嵌测试结果面板

将 toast 替换为**在服务商卡片内部直接展示的测试结果面板**，让反馈紧贴操作位置。

### 核心改动

#### 1. 状态升级 (`AgentModelsPage.tsx`)
- 将现有的 `testResultKey: { key, ok } | null` 替换为更丰富的 `modelTestResult` 状态：
```ts
{ providerId: string; modelId: string; ok: boolean; content?: string; error?: string } | null
```
- 保留按钮上的图标反馈（CheckCircle2 / XCircle），但延长显示时间与面板同步
- 移除 `runTestModel` 中的 `toast.success` / `toast.error` 调用

#### 2. 卡片内嵌结果面板
在每个 provider 卡片的 model 列表区域下方，当该卡片的某个模型有测试结果时，显示一个带动画的轻量面板：

- **成功态**：绿色左边框 + CheckCircle2 图标 + "测试通过" 标题 + 模型响应内容（等宽字体，最多显示 3 行）
- **失败态**：红色左边框 + XCircle 图标 + "测试失败" 标题 + 错误详情
- 面板在 model 列表和 footer 操作栏之间出现
- 滑入动画（`animate-in slide-in-from-top-2 fade-in duration-300`）
- 6 秒自动消失 + 手动关闭按钮
- 点击面板任意位置（非关闭按钮）可展开/收起完整内容

#### 3. 新增 i18n 文案
在 `zh/models.json` 和 `en/models.json` 的 `testDialog` 下新增：
- `modelTestPassed`: "测试通过"
- `modelTestFailed`: "测试失败"  
- `modelTestResponse`: "模型响应" / "Model Response"
- `modelTestError`: "错误详情" / "Error Details"

#### 4. 修改范围（3 个文件）
| 文件 | 改动 |
|------|------|
| `web/src/pages/agent-panel/pages/AgentModelsPage.tsx` | 状态替换、移除 toast、renderCard 内新增结果面板 JSX |
| `web/src/i18n/locales/zh/models.json` | 新增 4 个 i18n key |
| `web/src/i18n/locales/en/models.json` | 新增 4 个 i18n key |

### 不改动的部分
- sonner Toaster 全局配置不变（其他 toast 继续正常工作）
- 按钮图标反馈保留（CheckCircle2 / XCircle）
- Provider 级别的 fetchModels Dialog 保持不变
- 不引入新依赖

### 预期效果
- 用户在卡片左侧点击"测试"→ 结果面板直接在该卡片内弹出 → 视线无需移动
- 面板带有明显的颜色区分和动画，不易忽略
- 自动消失 + 可手动关闭，不阻塞后续操作
